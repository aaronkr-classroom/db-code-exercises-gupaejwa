개념퍼즐
- 가로
  1. 관계를 맺고 있는 두 개체 타입에서 한 개체 타입의 일부 개체 인스턴스만 다른 개체 타입의 개체 인스턴스와 연관되는 것 -> 부분참여
  6. ER 모델의 기본 요소 가운데 개체나 관계의 특성을 나타내는 요소 -> 속성
  8. 개체의 이름과 개체를 구성하는 속성들을 정의한 것 -> 개체타입
  9. 하나의 개체 타입이 동일한 관계 타입에 두 번 참여하는 것 -> 순환관계
  11. 각 속성이 가질 수 있는 값의 범위 -> 도메인
  12. ER 모델의 기본 요소 가운데 DB가 표현하려고 하는 유,무형의 정보 대상으로서, 현실 세계에 존재하면서 서로 구별할 수 있는 요소 -> 개체
- 세로
  2. ER 모델을 나타내는 그래픽 방식의 표현 방법 -> ER다이어그램
  3. 관계에 참여하는 하나의 개체에 대해 다른 개체가 몇 개 대응하는지를 나타내는 것 -> 카디널러티
  4. 특정 개체에 대해 여러 개의 값을 가질 수 있는 속성 -> 다중치속성
  5. 자체적으로 개체를 식별할 수 있는 기본키 속성을 갖지 못해서 독자적으로 존재할 수 없는 개체 -> 약한개체
  7. 관계로 연결된 개체 타입의 수 -> 관계차수
  10. ER 모델의 기본 요소 가운데 개체 간의 의미 있는 연결 또는 연관성을 표현하는 요소 -> 관계

연습문제
1. ER 모델을 구성하는 세 가지 기본 요소가 아닌 것은? 3)정보
2. 개체 타입과 개체 인스턴스의 차이점 :개체 타입은 같은 속성을 가지는 개체들의 집합, 개체 인스턴스는 개체 타입에 속하는 실제 개체
3. 다음은 쇼핑몰 DB 회원 개체에 필요한 속성들이다. 각 속성의 종류가 잘못 연결된 것은? 답에 대한 이유도 설명하시오. 2)휴대폰번호-상수 속성
4. 어떤 건물을 어떤 건축가가 건축했는지를 표현하는 ER 모델의 구성 요소 가운데 성격이 다른 것은? 답에 대한 이유도 설명하시오. 2)건축, 경력/가격/위치는 속성, 건축은 건물을 건축가가 건축했다의 관계이다.
5. 다음은 동아리 DB에서 회원 개체 타입과 행사 개체 타입 간의 참여하다 관계를 표현한 것이다. 적합한 관계 유형을 쓰시오. 회원 N, 행사 M
6. 다음은 경매 사이트 DB에서 작품 개체 타입과 입찰자 개체 타입 간의 낙찰되다 관계를 표현한 것이다. 적합한 관계 유형을 쓰시오. 작품 N, 입찰자 1
7. 어떤 컴퓨터가 어떤 회사에서 생산되었는지를 표현하는 ER 모델에서 다음 각각은 어떤 구성 요소에 해당하는지 답하시오.
   1)컴퓨터, 4)회사 -> 개체 2)모델명, 3)대표자, 6)생산년도, 7)메모라이즈, 8)주소 -> 속성, 5)생산 -> 관계
8. 은행에서 각 대출에 대한 상환 내역을 관리하려고 한다. 대출과 대출상환 간의 관계를 ER 다이어그랩으로 표현하시오. 단, 대출 개체 타입의 속성은 대출번호, 대출금, 대출일자이고, 대출상환 개체 타입의 속성은 상환날짜와 금액이며, 대출이 모두 상환되면 대출상환 정보도 함께 삭제한다.
